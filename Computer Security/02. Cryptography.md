# 02. Cryptography

## Uses of cryptography
- secure communications (HTTPS etc)
- File encryption (Encrypting file system on Microsoft NTFS, Trucypt)
- Data protection (CSS, AACS)

## Goals of cryptography
- Confidentiality
- Integrity
- non-repudiation
- authentication

## Cryptography terminology
1. plaintext = original unencrypted message
2. ciphertext = encrypted message
3. cipher = algorithm used to encrypt the message (transfrom from plaintext to ciphertext)
4. key = information used in the cipher 

## Cryptography System
Cryptography generally functions using two systems
- subsitution = substitute the letters (a to e, d to g etc..)
- transposition = moving the position of the letters

### Single-use key
- the key is only used to encrypt a message
- Eg: email encrypted with a different key

### Multi-use key
The key is used to encrypt multiple messages

### Julius Caeser
- maximum of 26 keys

### Monoalphabetic Cipher
- Works by shuffling letters
- Each letter has to have a unique substitute

### Vigenere Cipher
- Encryption works by mapping the plaintext character with key's character using the vigenere cipher table to find the final ciphertext character

### Rail Fence Cipher
Plaintext is written down as a sequence of diagonals and then read off as a sequence of rows

### Row transposition cipher
The Row Transposition Cipher (or Columnar Transposition) is a classical encryption technique that scrambles the order of letters in a message. It works by writing plaintext into a grid row by row, and then generating the ciphertext by reading the columns in an order dictated by a numerical key.
