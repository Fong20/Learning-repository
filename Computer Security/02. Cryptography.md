# 02. Cryptography
Cryptography is defined as the study of encryption principles/methods

## Uses of cryptography
- secure communications (HTTPS etc)
- File encryption (Encrypting file system on Microsoft NTFS, Trucypt)
- Data protection (CSS, AACS)

## Goals of cryptography
- Confidentiality
- Integrity
- non-repudiation
- authentication

## Cryptography terminology
1. plaintext = original unencrypted message
2. ciphertext = encrypted message
3. cipher = algorithm used to encrypt the message (transfrom from plaintext to ciphertext)
4. key = information used in the cipher. It must be private (only known to sender/receiver)
5. K = key
6. E = encrypt (public key)
7. D = decrypt (private key)
8. CT / C = cipher text
9. PT / P = plain text

## Cryptography System
Cryptography systems can be evaluated by

Type of encryption operation:
1. subsitution = substitute the letters (a to e, d to g etc..)
- A substitution cipher replaces one symbol with another.
- Monoalphabetic replaces the same symbol with the same another symbol.
- Polyalphabetic replaces the same symbol with different symbols at each occurrence.

2. transposition = moving the position of the letters

Number of keys used:
- Single-key: aka Symmetric, secret-key, private, or conventional cryptography (same key)
- Two-key: aka Asymmetric, two-key, or public cryptography (different keys)

Way in which plaintext is processed:
- Block (only one block at a time)
- Stream (process elements continuously)

## Symmetric cryptography
- utilizes a shared key for both encryption and decryption. The secret key is only known to the sender and receiver.
- Faster performance
- However, since the secret key is only known to the sender and receiver, a compromised key will allow the attacker to bypass the algorithm and view the data.

### Single-use key
- the key is only used to encrypt a message
- Eg: email encrypted with a different key

### Multi-use key
- The key is used to encrypt multiple messages
- The selection of “Algorithm” and “key” is critical in providing secure communication.

## Monoalphabetic Cipher
A monoalphabetic cipher is a type of substitution cipher where each plaintext letter is mapped to a fixed, unique ciphertext letter across the entire message. Regardless of where or how often a letter appears, it is always replaced by the same corresponding character.
  
### Caeser Cipher
- earliest and simplest methods of encryption.
- a type of monoalphabetic substitution cipher

#### Weakness
Because a letter always maps to the same character, monoalphabetic ciphers are easily cracked using frequency analysis. In English, letters like E, T, A, and O appear most frequently. By finding the most frequent characters in a secret message, a cryptanalyst can determine the original letters.

## Polyalphabetic Cipher
A polyalphabetic cipher is an encryption method that uses multiple substitution alphabets. Unlike monoalphabetic ciphers (such as a simple Caesar shift), which replace a letter with the same ciphertext character every time, polyalphabetic systems change the substitution rule based on the letter's position in the text.

### Vigenere Cipher
- Best known and simplest polyalphabetic substitution ciphers.
- In this scheme the set of related monoalphabetic substitution rules consists of the 26 Caesar ciphers with shifts of 0 through 25.
- To encrypt a message, a key is needed that is as long as the message and usually, the key is a repeating keyword. 
- Encryption works by mapping the plaintext character with key's character using the vigenere cipher table to find the final ciphertext character

### Rail Fence Cipher
Plaintext is written down as a sequence of diagonals and then read off as a sequence of rows

### Row transposition cipher
The Row Transposition Cipher (or Columnar Transposition) is a classical encryption technique that scrambles the order of letters in a message. It works by writing plaintext into a grid row by row, and then generating the ciphertext by reading the columns in an order dictated by a numerical key.

## Asymmetric cryptography
public-key/two-key/asymmetric cryptography involves the use of two keys: 

1. Public-key
- key which may be known by anybody, and can be used to encrypt messages, and verify signatures
- encryption keys, E, published or made public

2. Private-key
- key which is known only to the recipient, used to decrypt messages, and sign (create) signatures
- decryption keys, D, are kept secret

## Cryptoanalysis and Brute-force attack

### Cryptoanalysis
- Cryptoanalysis refers to the study of principles/methods of deciphering cipher text without knowing the key.
- Attack relies on the nature of the algorithm plus some knowledge of the general characteristics of the plaintext.
- Attack exploits the characteristics of the algorithm to attempt to deduce a specific plaintext or to deduce the key being used.

### Brute-force attack
Attacker tries every possible key on a piece of ciphertext until an intelligible translation into plaintext is obtained.
