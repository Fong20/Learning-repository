# 01. Introduction to Computer Security
- Cybersecurity is the practice of protecting critical systems and sensitive information from digital attacks

## Cybersecurity Subsets
1. Information security
- Informations security refers to the preservation of confidentiality, integrity, and availability of information
- Additionally, authenticity, accountability, nonrepudiation, and reliability can also be involved.

2. Network security
- Network security refers to the act of protecting networks and services from unwanted users, attacks, and intrusions.
  
## CIA Triad
CIA Triad consists of three aspects. All three aspects must be maintained and balanced.

1. Confidentiality = Ensures that the data can only be accessed by authorized user, Preserving unauthorized access to the data.
2. Integrity = Ensures that the data can only be modified only in a specified and authorized manner.
3. Availability = Ensures that only authorized users should be able to access data all the time. 

**Additional principles**
1. Authenticity = Authenticity ensures the user is who they say they are. 
2. Accountability = Accountability ensures that actions taken by an entity can be traced back to that entity. This is important for nonrepudiation, deterring breaches, finding faults, detecting and preventing intrusions, recovering from incidents, and taking legal action. Since perfect security isn't possible, it's crucial to trace security breaches to those responsible by keeping records of system activities for later analysis.
3. Non-repudiation: Assurance that the sender of information is provided with proof of delivery and the recipient is provided with proof of the sender’s identity, so neither can later deny having processed the information.

## Security Terms
1. Vulnerability = weakness in the system which can be exploited
2. Threat = malicious event which exploits the vulnerability
3. Attack = assault on the system security caused by the threat
4. control = action, device, procedure or technique that removes or reduces a vulnerability

Security policies = highest level of documentation that describes the security procedures or protocols to be followed by the company.

## OSI Security Architecture
- The ITU-T X.800 is an international standard that defines the Security Architecture for Open Systems Interconnection (OSI). It provides a systematic framework for securing network communications across all seven layers of the OSI model.
- Vendors have developed their security features on products and services based on this standard/framework.

**3 security asepcts of ITU-T X.800 standard** 
- Security attack
- Security mechanism
- Security service

### Security Attacks
There are two main types of attack

**1. Passive Attack**
- attack which monitors the but does not intend to cause disruption/harm to the system. It attempts to learn or make use of information from the system but does not affect system resources.
- These attacks are **difficult to detect because they do not involve any alteration of the data.**

**Two types of passive attack:**
1. Release of message contents: (monitoring the content to find sensitive information).
2. Traffic analysis: monitor traffic (looking for especial pattern such as pattern of messages, password length, location and identity of communicating hosts, …)

**2. Active Attack**
- Active attack refers to an attack which attempt to cause disruption/harm to the system resources or causea disruption.

**Four types of active attack:**
1. Masquerade : One entity pretends to be another entity (MITM).
2. Replay : Capture message and reply to get an authorized access.
3. Modification : Alter, reorder and delayed to an authorized access.
4. Denial of service: Prevents or inhibits the normal use or management of communications facilities.

**Active vs Passive Attacks**
- In passive attacks, the information does not modify, but in active attacks, the information will modify, delete, and add.
- An active attack will cause harm to the system, but a passive attack does not.
- Victims of an active attack are informed that their system is compromised, whereas victims of a passive attack are not informed.
- Active attacks are much more difficult to counter and prevent, whereas passive attacks are much easier to prevent.

### Security Services
The ITU-T X.800 standard divides security services into six categories to enhance the security of data processing and transferring.

1. Authentication : Assurance that the communicating entity is the one claimed.
2. Access control : Prevention of the unauthorized use of a resource.
3. Data confidentiality : Protection of data from unauthorized disclosure.
4. Data integrity : Assurance that data received is as sent by an authorized entity.
5. Nonrepudiation : Protection against denial by one of the parties in a communication.
6. Availability : Resource accessible/usable.

### Security Mechanism
1. Cryptographic algorithms (encipherment): The use of mathematical algorithms to transform data into a form that is not readily intelligible.
2. Data integrity: It covers a variety of mechanisms used to assure the integrity of a data unit or stream of data units.
3. Digital signature: Data appended to, or a cryptographic transformation of, a data unit that allows a recipient of the data unit to prove the source and integrity of the data unit and protect against forgery.
4. Authentication exchange: A mechanism intended to ensure the identity of an entity by means of information exchange.
5. Traffic padding: The insertion of bits into gaps in a data stream to frustrate traffic analysis attempts.
6. Routing control: Enables selection of particular physically or logically secure routes for certain data and allows routing changes, especially when a breach of security is suspected.
