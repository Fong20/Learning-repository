# 01. Introduction to Data Structure and Algorithms

## Data Structure
Data structure is an organized way to store, manage, and retrieve data efficiently, enabling effective access and modification depending on the use case.

### Types of Data Structures
1. Static Data Structure
- capacity is fixed at creation
- e.g. Array, Tuple

2. Dynamic Data Structure
- capacity is variable so it can expand or contract at any time
- e.g. Linked list, Binary tree.

## Algotithm 
Algorithm is defined as a procedure / formula to solve a problem

### Properties of Algorithms
- Correctness = Determines whether the algorithm produces the correct output for the given input
- Efficiency = how may steps does the algorithm take to solve the problem which affects the time / how much storage space does the algorithm take to solve the output. The faster the algorithm runs, the more likely the amount of storage space will be consumed.
- Applicability = Defines the factors where the algorithm can be applied such as the problem type, input constraint, performance, domain

Note: Correctness is always the highest priority compared to efficiency

### Types of analysis of algorithms
1. Experimental Analysis
- Research that derives its data by means of direct observation or experiment
- The results are based upon actual evidence
  
2. Theoretical Analysis
- Does not take into account of any hardware or software problems / limitations
- It uses mathematical tools such as Big O notation which focuses on identifying the algorithm efficiency amount of time taken based on the number of steps, based on the function of the input size, n.

## Loops

1. Single Loop 
for i in range(n):
  work()

2. Nested Loop
for i in range(n):
  for j in range(n):
    work()

3. Halving Loop
i = n
while i > 1:
    i //= 2

## Notations
There are several mathematical tools used to analyze algorithm efficiency

1. Big O Notation
- Worst Case Scenario (highest possible value)
- Upper bound or most amount of time required

2. Big Omega Notation
- Best Case Scenario (lowest possible value)
- Lower bound or the least amount of time required

3. Big Theta Notation
- Average Case Scenario
- Tight Bound, only if the best case and the worst case are the same.
