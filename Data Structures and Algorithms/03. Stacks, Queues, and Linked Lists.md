# 03. Stacks, Queues, and Linked Lists

## Pointers
Pointers are known as a variable that stores the memory location of another data

Referencing is known as the process of accessing the memory location of another data

## Linear Data Structure
- Data elements are arranged sequentially in single level
- e.g, Stacks, Queues, Arrays, Linked Lists

### Stacks
- Stack is a linear data structure that operates like a stack of dishes
- Works on Last In, First out (LIFO) principle, i.e. the last element added to the stack is the first one to be removed.
- Unlike arrays where we can extract the element from anywhere, we can only only extract the last element first due to LIFO principle
- Accepts any data type
- operate in O(1)
- e.g, Email inbox, Undo Steps, Browser History, etc.

#### Important Stack Operations
1. Push = add elements into the stack
2. Pop = remove elements from the stack and return the top element
3. Peek (Top): Returns the top element of the stack without removing it.
4. IsEmpty: Checks if the stack is empty.
5. Size: Returns the number of elements in the stack.

### Queues
- Queues is a linear data structure that operates like a queue or a line of people,
- operates as First in, First out, FIFO, i.e. the first element added to the queue is the first one to be removed.
- Accepts any data type
- operate in O(1)
- e.g, Task Scheduling, Data Buffering, Streaming, etc.

#### Important Queue Operations
1. Enqueue: Adds an element to the end (rear) of the queue.
2. Dequeue: Removes and returns the front element of the queue.
3. Front (or Peek): Returns the front element of the queue without removing it.
4. Rear (or Back): Returns the last element of the queue without removing it.
5. IsEmpty: Checks if the queue is empty.
6. Size: Returns the number of elements.

In Python, the deque implementation works similar to a queue, but with the additional of few operations such as  .popleft(), and .popright()

### Linked Lists
- A linked list is a dynamic linear data structure with a series of nodes connected by pointers / references
- operate in O(1), it does not depend on the number of nodes

#### Structure of a linked list
1. head (data)
2. next (pointer / reference)
3. Node = head (data) + next (pointer / reference)

<img width="2192" height="523" alt="image" src="https://github.com/user-attachments/assets/24f159cd-1326-4b0c-9f54-8b06809108af" />

#### Types of linked list
1. Singular linked list
- In a singular list, we are able to move to the next data but we are not able to move back to the previous data
- Requires 2 memory space, one to store the data, another for the pointer to store the memory location of the next data

<img width="535" height="281" alt="image" src="https://github.com/user-attachments/assets/f743c269-9974-47ba-ba97-6715ec4ed6cc" />

2. Circular linked list
- In the circular linked list, the last data points back to the first data (head). However, it is still unable to move back to the previous data as it does not have the memory address of the previous data
- - Requires 2 memory space, one to store the data, another for the pointer to store the memory location of the next data

<img width="704" height="329" alt="image" src="https://github.com/user-attachments/assets/3abe6ee4-154e-41cb-bb92-a26254572559" />

3. Doubly linked list
- Doubly linked list provides the most dynamic, where it is able to move to the next data, and back to the previous data thanks to the inclusion of both data's memory addresses.
- However, it takes up much more memory space. It requires 3 memory space, one to store the data, another to store the address of previous node, and another to store the adddress of the next node.

#### Linked List Operations
1. Traversal: To traverse all the nodes one after another.
2. Insertion : To add a node at the given position.
3. Deletion: To delete a node.
4. Searching: To search an element(s) by value.
5. Updating: To update a node.
6. Sorting: To arrange nodes in a linked list in a specific order.
7. Merging: To merge two linked lists into one.

- Arrays excel in accessing elements, but linked list excels at insertion and deletion of elements
- Linked list needs to access each element one by one until it finds the required element

## Non Linear Data Structure
- Data is arranged in Hierarchical Arrangement with Multiple Levels
- Eg: Trees, Graphs

### Trees
Non-linear, hierarchical data structure, a type of Graph*

#### Components of a tree
1. Root: The topmost node, the tree originates from root, and the only node with no parent. There is exactly one root in each tree.
2. Edge (Branch): The line that connects two nodes. If there are n nodes, there are n-1 edges.
3. Child: A descendant of a node, all nodes except root are child nodes.
4. Parent: The immediate predecessor of a node. Any node with one or more children is a parent node.
5. Siblings: A group of nodes with the same parent.
6. Leaf: A node with no child, also known as external or terminal nodes.
7. Internal Node: A node with at least one child node is called internal nodes.
