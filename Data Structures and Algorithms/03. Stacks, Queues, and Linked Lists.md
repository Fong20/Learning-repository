# 03. Stacks, Queues, and Linked Lists

## Pointers
Pointers are known as a variable that stores the memory location of another data


Referencing is known as the process of accessing the memory location of another data

## Linear Data Structure
- Data elements are arranged sequentially in single level
- e.g, Stacks, Queues, Arrays, Linked Lists

### Stacks
- Stack is a linear data structure that operates like a stack of dishes
- Works on Last In, First out (LIFO) principle, i.e. the last element added to the stack is the first one to be removed.
- Unlike arrays where we can extract the element from anywhere, we can only only extract the last element first due to LIFO principle
- Accepts any data type
- operate in O(1)
- e.g, Email inbox, Undo Steps, Browser History, etc.

#### Important Stack Operations
1. Push = add elements into the stack
2. Pop = remove elements from the stack and return the top element
3. Peek (Top): Returns the top element of the stack without removing it.
4. IsEmpty: Checks if the stack is empty.
5. Size: Returns the number of elements in the stack.

### Queues
- Queues is a linear data structure that operates like a queue or a line of people,
- operates as First in, First out, FIFO, i.e. the first element added to the queue is the first one to be removed.
- Accepts any data type
- operate in O(1)
- e.g, Task Scheduling, Data Buffering, Streaming, etc.

#### Important Queue Operations
1. Enqueue: Adds an element to the end (rear) of the queue.
2. Dequeue: Removes and returns the front element of the queue.
3. Front (or Peek): Returns the front element of the queue without removing it.
4. Rear (or Back): Returns the last element of the queue without removing it.
5. IsEmpty: Checks if the queue is empty.
6. Size: Returns the number of elements.

In Python, the deque implementation works similar to a queue, but with the additional of few operations such as  .popleft(), and .popright()

### Linked Lists
A linked list is a dynamic linear data structure with a series of nodes connected by pointers / references

#### Structure of a linked list
1. head (data)
2. next (pointer / reference)
3. Node = head (data) + next (pointer / reference)

#### Types of linked list
In a singular list, we are able to move to the next data but we are not able to move back to the previous data
